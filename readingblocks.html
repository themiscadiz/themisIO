Product
Overview

I prototyped a block puzzle game that builds sentences. The user interacts with screen-based tasks through physical objects. The physical objects are blocks that should be placed in an order that the task implies. When the child positions the blocks, this triggers an animation on the screen or the next steps to complete.

Design process

Ideate

The context of the use of this object is the relation between the instructor and one child. The user could be a child between the ages of 5 to 10, where some subjects are less complex or could be simplified.
The first exploration of the content is in relation to science, as the image shows below. However, to take advantage of the ‚Äòsequence‚Äô interaction that the system has, I also to work on building sentences, where the order is intrinsically important.

Paper Prototyping

I construct different length sentences with varying sizes of the block and box in order to test the affordance commodity of the sizes and the complexity of sentences.

User Flow

After testing possible interactions, I decided to stack block objects horizontally. The sequence in which the user inserts the block is intuitive and related to learning subjects such as reading and comprehension. With this in sight, I start to explore possible exercise examples and user flow for the interaction.

App User Flow

Error tolerances

Most essential for this prototype is to allow space for error tolerance.
If the sentence is not correct or incomplete, the animation cannot be fully completed.

Animation and Features

Hints and Rules

The animation is a visualization of the sentence that the user has to build. In the app, there are two buttons. The first button gives the user a hint as to what the sentence is about (as shown in the left image, below). The second button gives a quick summary of how to play the game (right image, below). To build more visibility into the system, I implement three circles on the top; these represents the number of tasks and the stage that the user is in.

Building the Prototype

System Diagram

I worked with Arduino using P5 Serial Control to send the reading to a P5*js Sketch

Functional Prototyping and User Testing

Having a functional prototype allowed me to test, with more reliability, how a user realistically interacts with the interface and object. After user testing, I laser cut a visible path (rectangle and arrows) demonstrating how to actually place the blocks in the box. This simple change immediately generated more visibility as to how the blocks should interact with the box.
